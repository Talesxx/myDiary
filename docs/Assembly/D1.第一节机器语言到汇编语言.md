# 第一节 机器语言到汇编语言

## 一、机器语言的局限性

### 1.1 机器语言的特点

- 由二进制代码（0和1）组成
- 计算机可以直接执行
- 执行效率最高

### 1.2 机器语言的缺点

- **难以阅读和理解**：全是0和1，没有语义
- **难以调试**：出错时很难定位问题
- **可移植性差**：不同CPU架构的机器语言完全不同
- **容易出错**：手动编写二进制代码容易出错

## 二、汇编语言的诞生

### 2.1 什么是汇编语言

汇编语言是机器语言的符号化表示，用助记符（Mnemonics）代替二进制操作码。

### 2.2 汇编语言的优势

- **可读性好**：用英文单词或缩写表示指令
- **易于编写**：减少出错概率
- **可调试性强**：便于理解程序逻辑
- **与机器语言一一对应**：效率仍然很高

### 2.3 汇编语言的应用场景

#### 2.3.1 适用场景

- 操作系统内核开发
- 嵌入式系统编程
- 性能要求极高的模块
- 设备驱动程序
- 逆向工程和漏洞分析

#### 2.3.2 不适用场景

- 大型应用程序开发
- 快速原型开发
- 跨平台软件开发

## 三、以16位为例说明8086的总线宽度

### 3.1 总线

- 数据总线：用于传输数据
- 地址总线：用于传输地址
- 控制总线：用于传输控制信号

#### 3.1.1 地址总线

- 地址总线为n时，对应寻址空间为2^n
- 用于指定存储单元的地址，决定了寻址空间的大小
- 例如，16位地址总线可以寻址2^16个存储单元（通常一个存储单元为1字节8bit）

#### 3.1.2 数据总线

- CPU与内存或其它器件之间的数据传送是通过数据总线来进行的
- 数据总线的宽度决定了CPU和外界的数据传送速度
- 数据总线宽度决定一次传送的数据量，一个通道为1位，一次即1bit数据
- 例：向内存中写入数据89D8H时的数据传送
- 注意：1字节（1B）= 8位（8bit）

#### 3.1.3 控制总线

- 控制总线是一些不同控制信号线的集合
- CPU通过控制总线对外部设备进行控制
- 控制总线决定了CPU对外部设备的控制能力

### 3.2 数据如何表示

- 0100 1001 1101 1000B 二进制
- 44730O 八进制
- 49D8H 十六进制
- 18904D 十进制

## 四、机器语言与汇编语言的对应关系

### 4.1 操作码的映射

| 机器语言（二进制） | 汇编语言（助记符） | 功能描述 |
| ------------------ | ------------------ | -------- |
| 0000               | NOP                | 空操作   |
| 0001               | MOV                | 数据传送 |
| 0010               | ADD                | 加法     |
| 0011               | SUB                | 减法     |
| 0100               | JMP                | 跳转     |

### 4.2 汇编语言示例

```assembly
MOV AX, 5    ; 向寄存器AX加载数据5
ADD AX, 3    ; AX寄存器加3
JMP 10       ; 跳转到地址10
```

## 五、汇编语言的基本组成

### 5.1 指令（Instructions）

- 操作码（Opcode）：如 MOV、ADD、SUB
- 操作数（Operands）：寄存器 register、内存地址 [address]、立即数 immediate

### 5.2 伪指令（Directives）

- 指示汇编器如何处理程序
- 不生成机器码
- 示例：ORG、EQU、DB、DW

### 5.3 标号（Labels）

- 表示内存地址
- 便于程序跳转和数据访问
- 示例：START、LOOP、END

### 5.4 物理地址与段地址

- 在8086汇编中，"段地址 × 16 + 偏移地址 = 物理地址" CS:IP，其中CS是代码段寄存器，IP是指令指针寄存器
- 8086其CPU是16位的，所以地址总线为16位，对应寻址空间为2^16个存储单元，其寄存器宽度为16位（16bit，2字节2B）
- CS和IP是不可直接访问的，只能通过指令来操作。在编程中不用考虑段地址，只需要考虑偏移地址
- 在编程中，你几乎感觉不到CS+IP的存在，因为汇编器（编译器）帮你把这些脏活累活都干了

```assembly
    MOV AX, 1
    JMP label    ; 你只关心跳转到label这个标签，它计算出label标签距离当前指令的偏移地址，修改IP的值
label:
    ADD AX, 2
```

- 在现代计算机中是32位地址总线或者64位地址总线，对应寻址空间为2^32或2^64个存储单元，其寄存器宽度为32位或64位

### 5.5 单位

- **位（Bit）**：最小的二进制单位，0或1
- **字节（Byte）**：8位，用于存储和处理数据的基本单位
- **字（Word）**：16位，用于存储和处理数据的基本单位（汇编里的Word = 永远固定16位，2字节2B，16bit）
- **机器字长（Machine Word Length）**：指机器一次可以处理的最大数据长度，与计算机位数相同

## 六、汇编过程

### 6.1 汇编器的作用

将汇编语言源程序翻译成机器语言目标程序。

### 6.2 汇编步骤

1. **词法分析**：识别助记符、标号、操作数
2. **语法分析**：检查指令格式
3. **符号表建立**：记录标号地址
4. **代码生成**：将助记符转换为机器码
5. **链接**：组合多个目标文件

## 七、寄存器的使用

### 7.1 通用寄存器（X86下的寄存器）

- AX、BX、CX、DX：通用数据寄存器
  - AX：累加器寄存器，用于累加和存储数据
  - BX：基址寄存器，用于存储基址
  - CX：计数寄存器，用于循环计数
  - DX：数据寄存器，用于存储数据
- SI、DI：源变址和目的变址寄存器（存放偏移地址）
- SP、BP：栈指针和基址指针
- CS、IP：代码段寄存器和指令指针寄存器

### 7.2 段寄存器

- CS：代码段寄存器
- DS：数据段寄存器
- ES：附加段寄存器
- SS：栈段寄存器

> 在32位CPU中，CS、DS、SS等段寄存器里存放的不再是直接的段基址，但依然是16位，而是一个叫"段选择子(Selector)"的索引值（相当于一个数组下标）
> 在64位CPU中，段寄存器被取消了，除了FS和GS这两个特殊的段寄存器外，其他的段寄存器（CS, DS, ES, SS）的段基址被CPU强制设为0
> 彻底扁平化：线性地址 = 0 + 偏移地址，FS和GS的特殊用途

### 7.3 标志位

- ZF：零标志位，当结果为0时为1，否则为0
- CF：进位标志位，当结果为进位时为1，否则为0
- AF：辅助进位标志位，当结果为辅助进位时为1（低4位进位），否则为0
- SF：符号标志位，当结果为负数时为1（计算的最高位是否为1），否则为0
- OF：溢出标志位，当结果溢出时为1，否则为0
- PF：奇偶标志位，当结果为偶数时为1（结果的二进制数的所有位中1的个数为偶数），否则为0

## 八、常用汇编指令示例

### 8.1 MOV指令

- `mov target,source` ; target表示目标寄存器或内存地址，source表示源寄存器或立即数
- `mov reg/mem,imm` ; mem表示内存，imm表示立即数（数据），seg表示段寄存器，reg表示寄存器
  - 示例：`mov AX, 10H` ; 将立即数0010H移动到AX寄存器（立即数正数自动补0，负数高位全部补1，直到与target数的位数相同）
  - 示例：`mov DS, AX` ; 将AX寄存器的值移动到DS数据段寄存器
  - 示例：`mov [si], 1010H` ; 将1010H的值移动到DS:[SI]内存单元中，DS是数据段寄存器
- `mov reg/mem/seg,reg`
- `mov reg/seg,mem`
- `mov reg/mem,seg`

#### 8.1.1 寻址方式

- 立即数寻址
  - 立即数寻址：将立即数直接作为操作数，不使用寄存器或内存地址
  - 示例：`mov AX, 10H`
- 寄存器寻址
  - 寄存器寻址：将寄存器作为操作数
  - 示例：`mov AX, BX`
- 存储器寻址（直接寻址）
  - 直接寻址：将内存地址直接作为有效地址
  - 示例：`mov AX, [1234H]`

以下是8086特有的内存寻址组合（基于基址和变址寄存器）：

- 存储器寻址（寄存器间接寻址）
  - 寄存器间接寻址：操作数的有效地址存放在特定的寄存器中，可用寄存器：仅限BX, BP, SI, DI
  - 示例：`mov AX, [BX]`
- 存储器寻址（基址寻址）
  - 基址寄存器+偏移量寻址：将基址寄存器的值作为有效地址，偏移量作为有效地址的偏移量
  - 示例：`mov AX, [BX + 100]`
- 存储器寻址（变址寻址）
  - 变址寄存器+偏移量寻址：将变址寄存器的值作为有效地址，偏移量作为有效地址的偏移量
  - 示例：`mov AX, [SI + 100]`
- 存储器寻址（基址变址寻址）
  - 基址寄存器+变址寄存器+偏移量寻址：将基址寄存器的值作为有效地址，变址寄存器的值作为偏移量，偏移量作为有效地址的偏移量
  - 示例：`mov AX, [BX + SI + 100]`

> **注意事项：**
> 1. target不能是CS（代码段寄存器）
> 2. target和source不能同时为内存数、段寄存器（CS\DS\ES\SS\FS\GS）
> 3. target等于source的长度，才能将数据传送，立即数则是小于等于target的位数即可
> 4. 不能将立即数传送给段寄存器
> 5. target和source必须类型匹配，比如，要么都是字节，要么都是字或者都是双字等
> 6. 由于立即数没有明确的类型，所以将立即数传送到target时，系统会自动将立即数零扩展到与target数的位数相同，再进行传送。有时，需要用BYTE PTR、WORD PTR明确指出内存地址的位数

![1780417007395](images/D1.第一节机器语言到汇编语言/1780417007395.png)

- `movzx target,source`（不能是立即数）; target必须大于source的长度，将source的值零扩展到与target数的位数相同，再传送。target必须是字节或字，其他和mov指令相同
- `movsx target,source`（不能是立即数）; target必须大于source的长度，将source的值高位全补1扩展到与target数的位数相同，再传送。target必须是字节或字

### 8.2 加法指令

- `ADD target,source` ; target = target + source，target表示目标寄存器或内存地址，source表示源寄存器或立即数。相加如果进位，进位标志位CF为1，否则为0
- `ADD reg,imm/mem/reg`
- `ADD mem,imm/reg`
- `ADC target,source` ; target = target + source + CF，CF是进位标志位
- `ADC reg,imm/mem/reg`
- `ADC mem,imm/reg`
- `INC target` ; target = target + 1，即使发生进位也不改变标志位CF
- `INC reg/mem`

#### 8.2.1 示例

- `ADD AX, BX`      ; AX = AX + BX
- `ADD AX, 10H`      ; AX = AX + 10H
- `ADD AX, [1000]`  ; AX = AX + 内存地址1000的值
- `ADC AX, BX`      ; AX = AX + BX + CF
- `INC AX`          ; AX加1

### 8.3 减法指令

- `SUB target,source` ; target = target - source，target表示目标寄存器或内存地址，source表示源寄存器或立即数。相减如果借位，借位标志位CF为1，否则为0
- `SUB reg,imm/mem/reg`
- `SUB mem,imm/reg`
- `SBB target,source` ; target = target - source - CF，target表示目标寄存器或内存地址，source表示源寄存器或立即数。相减如果借位，借位标志位CF为1，否则为0
- `SBB reg,imm/mem/reg`
- `SBB mem,imm/reg`
- `DEC target` ; target = target - 1，即使发生借位也不改变标志位CF
- `DEC reg/mem`

#### 8.3.1 示例

- `SUB AX, BX`      ; AX = AX - BX
- `SUB AX, 10H`      ; AX = AX - 10H
- `SUB AX, [1000]`  ; AX = AX - 内存地址1000的值
- `SBB AX, BX`      ; AX = AX - BX - CF
- `DEC AX`          ; AX减1

### 8.4 比较指令

- `CMP target,source` ; 不修改target的值，target表示目标寄存器或内存地址，source表示源寄存器或立即数。相减如果借位，借位标志位CF为1，否则为0
- `CMP reg,imm/mem/reg`
- `CMP mem,imm/reg`

#### 8.4.1 无符号比较

- 情况AX = BX
  - AX = BX，CF = 0，ZF = 1
  - 对应跳转指令：JE/JZ (Jump if Equal/Zero)
- 情况AX > BX
  - AX > BX，CF = 0，ZF = 0
  - 对应跳转指令：JA (Jump if Above) / JAE (Jump if Not Below or Equal)
- 情况AX < BX
  - AX < BX，CF = 1，ZF = 0
  - 对应跳转指令：JB (Jump if Below) / JC (Jump if Carry) / JNAE (Jump if Not Above or Equal)

#### 8.4.2 有符号比较

- 情况AX = BX
  - AX = BX，ZF = 1
  - 对应跳转指令：JE (Jump if Equal)
- 情况AX > BX
  - 情况一：正常的大数减小数，结果为正，SF = 0，且没有溢出 OF = 0
  - 情况二：正数减去一个极大的负数，导致正溢出变成了负数，此时 SF = 1，但 OF = 1
  - 标志位特征：SF = OF（两者同为0或同为1），且 ZF = 0
  - 对应跳转指令：JG (Jump if Greater) / JNLE (Jump if Not Less or Equal)
- 情况AX < BX
  - 情况一：正常的小数减大数，结果为负，SF = 1，且没有溢出 OF = 0
  - 情况二：负数减去一个极大的正数，导致负溢出变成了正数，此时 SF = 0，但 OF = 1
  - 标志位特征：SF != OF
  - 对应跳转指令：JL (Jump if Less) / JNGE (Jump if Not Greater)

### 8.5 与、或、异或、非指令

#### 8.5.1 与指令

- `AND target,source` ; target = target & source 按位与，target表示目标寄存器或内存地址
- `AND reg,imm/mem/reg`
- `AND mem,imm/reg`
- `AND AX, BX`     ; AX = AX & BX
- `AND AX, 10H`     ; AX = AX & 10H
- `AND AX, [1000]`  ; AX = 内存地址1000的值 & AX

#### 8.5.2 或指令

- `OR target,source` ; target = target | source 按位或，target表示目标寄存器或内存地址
- `OR reg,imm/mem/reg`
- `OR mem,imm/reg`
- `OR AX, BX`     ; AX = AX | BX
- `OR AX, 10H`     ; AX = AX | 10H
- `OR AX, [1000]`  ; AX = 内存地址1000的值 | AX

#### 8.5.3 异或指令

- `XOR target,source` ; target = target ^ source 按位异或，target表示目标寄存器或内存地址
- `XOR reg,imm/mem/reg`
- `XOR mem,imm/reg`
- `XOR AX, BX`     ; AX = AX ^ BX
- `XOR AX, 10H`     ; AX = AX ^ 10H
- `XOR AX, [1000]`  ; AX = 内存地址1000的值 ^ AX

#### 8.5.4 非指令

- `NOT target` ; target = ~target 按位取反，target表示目标寄存器或内存地址
- `NOT reg/mem`
- `NOT AX`        ; AX = ~AX
- `NOT [1000]`    ; 内存地址1000的值 = ~内存地址1000的值

### 8.6 JMP指令

- 语法：JMP 目标地址
- 功能：无条件跳转到目标地址，修改的是IP指令指针寄存器
- 示例：JMP label/address  ; 跳转到标签label
- JZ label/address  ; 如果ZF标志位为1，跳转到标签label
- JNZ label/address  ; 如果ZF标志位为0，跳转到标签label
- JE label/address  ; 如果ZF标志位为1，跳转到标签label
- JNE label/address  ; 如果ZF标志位为0，跳转到标签label
- JG label/address  ; 如果SF标志位为0，跳转到标签label
- JL label/address  ; 如果SF标志位为1，跳转到标签label
- JAE label/address  ; 如果ZF标志位为0，跳转到标签label
- JBE label/address  ; 如果ZF标志位为1，跳转到标签label
- JNA label/address  ; 如果CF标志位为0，跳转到标签label
- JNAE label/address  ; 如果CF标志位为1，跳转到标签label

> 需要注意的是，IP寄存器的位宽会随着CPU工作模式的改变而扩展：
> - 在16位实模式下：它被称为IP (Instruction Pointer)，是16位的
> - 在32位保护模式下：它被扩展为EIP (Extended Instruction Pointer)，是32位的
> - 在64位长模式下：它被进一步扩展为RIP (Extended Instruction Pointer)，是64位的
> 无论名称和位宽如何变化，JMP指令的本质操作始终是修改这个指向"下一条指令"的指针寄存器

### 8.7 程序框架

```asm
; ==========================================
; 1. 数据段 (Data Segment)
; 用于定义和初始化程序中使用的变量、常量等
; ==========================================
DATA SEGMENT
    ; 在这里定义你的变量，例如：
    MSG DB 'Hello, Assembly!', '$'   ; 定义一个字符串，以'$'结尾供DOS显示
    NUM DW 1234H                     ; 定义一个字(16位)数据
DATA ENDS

; ==========================================
; 2. 代码段 (Code Segment)
; 包含程序的执行逻辑和控制流
; ==========================================
CODE SEGMENT
    ASSUME CS:CODE, DS:DATA          ; 告诉编译器CS指向CODE，DS指向DATA

START:                               ; 程序入口点标签
    ; --- 初始化数据段寄存器 ---
    MOV AX, DATA                     ; 将数据段的段基址装入AX
    MOV DS, AX                       ; 再将AX的值传给DS（不能直接MOV DS, DATA）

    ; --- 你的主要程序逻辑写在这里 ---
    ; 例如：
    ; LEA DX, MSG                    ; 取字符串偏移地址
    ; MOV AH, 09H                    ; DOS功能号：显示字符串
    ; INT 21H                        ; 触发中断

    ; --- 正常退出程序并返回DOS ---
EXIT_PROGRAM:
    MOV AH, 4CH                      ; DOS功能号：终止程序
    INT 21H                          ; 触发中断，返回操作系统
CODE ENDS                            ; 代码段结束
END START                            ; 伪指令：标记程序结束，并指定START为入口点
```

### 8.8 伪指令

#### 8.8.1 数据定义

- [变量名] [数据类型] [数据值]
- DUP 重复定义指令：使用方法 [变量名] [数据类型] [重复次数（可以为变量名）] DUP [数据值]
- EQU 定义宏指令：使用方法 [变量名] EQU [数据值]（只能定义常量）
- DB DW DD DS CS SS ES FS GS 数据段定义指令
- SEGMENT 段定义指令
- ENDS 段结束指令
- END 程序结束指令
- CALL 调用指令
- RET 返回指令
- INT 中断指令
